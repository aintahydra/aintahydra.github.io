Prelude

installation

sudo apt install golang-go

set GOROOT($HOME/go) and GOPATH($GOROOT/bin). GOPATH isn’t used anymore. Since Go 1.11, go to the project directory, and execute go mod init github.com/_USER_/_REPO__.

Terms

  • Package (~= directory): compiled sources within a package can access each other. To access the other package, the package should be imported.
  • Module: inter-related packages. go.mod declares packages that are included a module.

module: an example

Ref: https://stackoverflow.com/questions/61845013/package-xxx-is-not-in-goroot-when-building-a-go-project

Say a project that has following directories:

project001/
 - src
  - go.mod
  - module001
   - externmod.go
    - externmod.g
  - main.go
  - module001

How to create them?

  1. create directories: mkdir -p project001/src/module001.
  2. create go.mod, by (1))move to project001/src, and (2)do go mod init module001.
  3. check out the contents of go.mod:
module module001
go 1.18
  1. create src/main.go:
package main

import (
	"exe/externmod"
)

func main() {
	externmod.func1()
}
5. create module001/externmod/externmod.go

package module001

import ( “fmt” )

func func1() { fmt.Printf(“I am func1 @ externmod\n”) }

Currently, the directory structures look like:

src$ tree . . ├── go.mod ├── main.go └── module001 └── externmod.go


Now, go to src/, do `go run main.go`.
